1. ОБЩЕЕ ОПИСАНИЕ ПРОЕКТА

1.1 Назначение

BI Analytics Dashboard — веб-приложение для аналитики продаж, интегрированное с Bitrix24 CRM. Приложение отображает ключевые KPI, воронки продаж, эффективность менеджеров и каналов привлечения в реальном времени.

1.2 Текущий стек технологий

КомпонентТехнология
FrontendHTML5 + Vanilla JS + CSS3
BackendNode.js + Express.js
APIBitrix24 REST API
Хранилище токеновJSON-файл (db.json)
HTTP-клиентAxios
ДеплойPM2 + Rsync на VPS
ШрифтInter (Google Fonts)

    • 4.2 🎓 Интерактивный онбординг (Interactive Onboarding)

    4.2.1 Описание

    Пошаговый интерактивный тур по дашборду для новых пользователей. Запускается автоматически при первом посещении и доступен для повторного запуска из меню.

    4.2.2 Сценарий онбординга (шаги)

    ШагЭлементПодсказка
    1Весь экран«Добро пожаловать в BI Analytics! Давайте познакомимся с основными возможностями дашборда.»
    2KPI-карточки«Здесь отображаются ключевые метрики: выручка, количество лидов, конверсия и активные сделки. Данные обновляются в реальном времени.»
    3Фильтр периода«Переключайте период для анализа: день, неделя или месяц. Данные автоматически пересчитаются.»
    4Воронка продаж«Воронка показывает, сколько сделок находится на каждом этапе. Выберите конкретную воронку или смотрите все сразу.»
    5Выбор воронки (select)«Здесь можно выбрать конкретную воронку продаж из вашего Битрикс24.»
    6Источники обращений«Кольцевая диаграмма показывает распределение лидов по каналам привлечения.»
    7Эффективность менеджеров«Рейтинг менеджеров по ключевым показателям. Сортируйте по конверсии, выручке или количеству сделок.»
    8Таблица сделок«Детальная таблица активных сделок с цветовой индикацией сроков. Красный = сделка "зависла".»
    9Каналы«Сравнивайте эффективность маркетинговых каналов по конверсии или количеству лидов.»
    10Кнопка обновления«Нажмите, чтобы обновить все данные с Bitrix24.»
    11Боковое меню«Используйте навигацию для переключения между разделами.»
    12Завершение«Готово! Теперь вы знаете основы. Нажмите ❓ на любом виджете для получения подсказки.»

    4.2.3 UI/UX онбординга

  • Overlay — затемнённый фон с перфорацией (spotlight) вокруг целевого элемента
  • Tooltip-облачко — появляется рядом с элементом (сверху/снизу/сбоку, автопозиционирование)
  • Кнопки навигации: «Назад», «Далее», «Пропустить тур», индикатор прогресса (шаг X из Y)
  • Анимация перехода — плавное перемещение spotlight между элементами (300ms)
  • Пульсирующая точка — на целевом элементе для привлечения внимания
  • Прогресс-бар — визуальный индикатор прохождения тура в верхней части подсказки

    • 4.2.4 Техническая реализация

  • Собственная реализация без внешних библиотек (чистый JS + CSS)
  • Хранение статуса прохождения: localStorage.setItem('onboarding_completed', 'true')
  • CSS: .onboarding-overlay, .onboarding-spotlight, .onboarding-tooltip
  • Автоматический скролл к целевому элементу при переходе между шагами
  • Адаптация под Bitrix24 iframe-режим (пропуск шагов, связанных с sidebar)
  • Повторный запуск: кнопка «Пройти обучение» в разделе «Настройки» или по ?tour=true в URL

    • 4.2.5 Условия запуска

  • Автозапуск: если localStorage не содержит onboarding_completed
  • Ручной запуск: пункт меню «Настройки → Пройти обучение заново»
  • Пропуск в iframe: если bitrix-mode, скрывать шаги, связанные с sidebar
  • Задержка старта: 1.5 секунды после загрузки данных, чтобы дашборд успел отрисоваться

    • 4.3 ❓ Иконки-подсказки на каждом виджете (Help Tooltips)

    4.3.1 Описание

    На каждом виджете (карточке, графике, таблице) в правом верхнем углу заголовка размещается иконка ❓ в кружке. При нажатии — всплывающее окно с пояснением к данному виджету.

    4.3.2 Список виджетов и тексты подсказок

    ВиджетТекст подсказки
    KPI «Выручка»«Сумма всех успешно закрытых сделок (стадия "Выиграна") за выбранный период. Валюта: ₽. Процент изменения рассчитан относительно аналогичного предыдущего периода.»
    KPI «Новые лиды»«Количество лидов, созданных в CRM за выбранный период. Лид — первичное обращение потенциального клиента.»
    KPI «Конверсия»«Процент сделок со статусом "Выиграна" от общего числа сделок за период. Формула: (выигранные / все сделки) × 100%.»
    KPI «Сделки в работе»«Количество активных сделок (не закрытых и не проигранных). Цветовая индикация: 🔴 горячие (близки к закрытию), 🟡 в процессе, ⚪ стагнирующие (без активности).»
    Воронка продаж«Визуализация распределения сделок по этапам выбранной воронки. Ширина полосы пропорциональна количеству сделок на этапе. Проценты между этапами показывают конверсию перехода.»
    Источники обращений«Распределение лидов по каналам привлечения. Данные берутся из поля "Источник" в карточке лида CRM. Переключайте табы для просмотра принятых и непринятых обращений.»
    Эффективность менеджеров«Рейтинг менеджеров отдела продаж. По умолчанию сортировка по конверсии. Выручка менеджера = сумма его выигранных сделок за период.»
    Таблица сделок«Список активных сделок с указанием менеджера, этапа и длительности нахождения в работе. Цветная метка срока: зелёный (< 7 дн.), синий (7-14), жёлтый (14-30), красный (> 30).»
    Эффективность каналов«Сравнительный анализ маркетинговых каналов. "Конверсия" — процент лидов, ставших сделками. "Лиды" — абсолютное число обращений из канала.»

    • 4.3.4 Техническая реализация

    7. НЕФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

    7.1 Производительность

  • Время загрузки дашборда: < 3 секунды (с учётом API Bitrix24)
  • Все API-запросы Bitrix24 — параллельно (Promise.all)
  • Кеширование данных на фронте (5 минут)
  • Lazy-loading для страниц «Аналитика», «Обращения»

    • 7.2 Совместимость

  • Браузеры: Chrome 90+, Firefox 90+, Safari 14+, Edge 90+
  • Bitrix24 Cloud iframe (desktop и мобильное приложение)
  • Разрешения: от 375px (мобильный) до 4K

    • 7.3 Безопасность

  • HTTPS обязателен
  • Токены не передаются на фронтенд
  • CORS: ограничение по домену Bitrix24
  • Rate limiting на API-эндпоинты
  • Валидация входных параметров (sanitize domain, categoryId)

    • 7.4 Логирование

  • Структурированный лог на бэкенде: [Module] Message
  • Логирование всех API-вызовов к Bitrix24
  • Логирование ошибок с контекстом (домен, метод, параметры)
