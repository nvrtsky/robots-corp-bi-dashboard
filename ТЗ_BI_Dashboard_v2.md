## 1. ОБЩЕЕ ОПИСАНИЕ ПРОЕКТА

### 1.1 Назначение
BI Analytics Dashboard — веб-приложение для аналитики продаж, интегрированное с Bitrix24 CRM. Приложение отображает ключевые KPI, воронки продаж, эффективность менеджеров и каналов привлечения в реальном времени.

### 1.2 Текущий стек технологий
| Компонент | Технология |
|-----------|-----------|
| Frontend | HTML5 + Vanilla JS + CSS3 |
| Backend | Node.js + Express.js |
| API | Bitrix24 REST API |
| Хранилище токенов | JSON-файл (db.json) |
| HTTP-клиент | Axios |
| Деплой | PM2 + Rsync на VPS |
| Шрифт | Inter (Google Fonts) |

- **VPS:** `79.174.77.28`, порт `3010`

### 2.5 Источники обращений
- Donut-диаграмма (SVG, `stroke-dasharray`)
- Легенда с прогресс-барами по каждому источнику
- Табы: Принятые / Непринятые
- Блок статистики непринятых заявок
- Динамическая гидрация из `crm.status.list` (SOURCE)

### 2.6 Эффективность менеджеров
- Ранжированный список (Top-5)
- Аватар (инициалы), имя, количество сделок
- Горизонтальный прогресс-бар по конверсии
- Общая выручка менеджера
- Медали: 🥇🥈🥉 для топ-3
- Сортировка: по конверсии / по выручке / по сделкам

### 2.7 Таблица сделок в работе
- Колонки: Сделка, Клиент, Менеджер, Этап, Сумма, В работе
- Цветовая индикация длительности: < 7 дн. (зелёный) / 7-14 дн. (синий) / 14-30 дн. (жёлтый) / > 30 дн. (красный)
- Summary-блок: Всего сделок, Общая сумма, Средний срок
- Легенда длительностей

### 2.8 Эффективность каналов
- Горизонтальные бары с цветовыми иконками
- Каналы: Органический трафик, Контекстная реклама, Соцсети, Email, Партнёры
- Переключатель метрик: Конверсия / Лиды
- Динамическая генерация из данных API

### 2.9 Серверная часть (Backend)

#### API-эндпоинты:
| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/` | Отдача index.html |
| POST | `/` | Отдача index.html (Bitrix24 POST) |
| POST | `/api/bitrix24/install` | OAuth Install Handler |
| GET | `/api/stats/dashboard` | Главный эндпоинт дашборда |
| GET | `/api/funnels` | Список воронок |
| GET | `/api/health` | Health check |

#### Параметры `/api/stats/dashboard`:
- `dateFrom` — дата начала периода
- `dateTo` — дата конца периода
- `domain` — домен Bitrix24 портала
- `categoryId` — ID воронки (необязательно, `all` = все)

#### Bitrix24 API вызовы (параллельные):
1. `crm.deal.list` (STAGE_ID = WON) — выручка
2. `crm.lead.list` — лиды и источники
3. `crm.deal.list` (!WON, !LOSE) — сделки в работе
4. `crm.deal.list` (все) — данные воронки
5. `crm.dealcategory.stage.list` — стадии воронки
6. `crm.status.list` (SOURCE) — словарь источников
7. `crm.dealcategory.list` — категории воронок
8. `user.get` — имена менеджеров

### 2.10 Авторизация и токены
- OAuth-установка через POST `/api/bitrix24/install`
- Хранение токенов в `db.json` (файловая БД)
- Автоматический refresh токена при `expired_token`
- Fallback-цепочка определения домена: `req.query` → `storage` → `.env` → hardcode

### 2.11 Mock-данные
- Модуль `mock-data.js` для локальной разработки без Bitrix24
- Генерация реалистичных данных: воронки по категориям, менеджеры, сделки

## 3. ДОРАБОТКИ СУЩЕСТВУЮЩЕГО ФУНКЦИОНАЛА

### 3.1 Навигация по страницам
**Текущее состояние:** Пункты меню (Аналитика, Менеджеры, Обращения, Настройки) не работают — только переключают `active` класс.

**Требования:**
- Реализовать полноценную навигацию (SPA-роутинг или секции с переключением видимости)
- Каждый пункт меню должен отображать соответствующий контент

#### 3.1.1 Страница «Аналитика»
- Графики динамики за период (линейные/столбиковые)
- Сравнение периодов (текущий vs предыдущий)
- Тренд-анализ: выручка, лиды, конверсия за месяцы
- Экспорт данных в CSV/Excel

#### 3.1.2 Страница «Менеджеры»
- Детальная карточка каждого менеджера
- Персональная воронка менеджера
- История сделок менеджера
- KPI-цели и факт/план
- Сравнительный анализ менеджеров

#### 3.1.3 Страница «Обращения»
- Лента обращений (лидов) в реальном времени
- Фильтрация по источнику, статусу, менеджеру
- Быстрый просмотр карточки лида
- Статистика по обработке обращений

#### 3.1.4 Страница «Настройки»
- Настройки подключения к Bitrix24 (домен, перевыпуск токена)
- Настройки виджетов дашборда (показать/скрыть карточки)
- Управление KPI-целями (план выручки, план лидов)
- Конфигурация уведомлений

### 3.2 Адаптивность и отзывчивый дизайн
**Текущее состояние:** CSS содержит медиа-запросы, но не все компоненты адаптивны.

**Требования:**
- **Мобильная версия** (< 768px): сайдбар → гамбургер-меню, KPI-карточки в 1 колонку, таблица с горизонтальным скроллом
- **Планшет** (768–1024px): KPI 2×2, графики в 1 колонку
- **Desktop** (> 1024px): текущий layout
- Корректная работа в Bitrix24 iframe на мобильных устройствах

### 3.3 Фильтрация и выбор дат
**Текущее состояние:** Только предустановленные периоды (день/неделя/месяц).

**Требования:**
- Добавить календарь выбора произвольного диапазона дат (date range picker)
- Быстрые пресеты: «Вчера», «Этот квартал», «Прошлый квартал», «Этот год»
- Сравнение с предыдущим аналогичным периодом

### 3.4 Обработка ошибок и состояний загрузки
**Текущее состояние:** Минимальная обработка ошибок, нет индикаторов загрузки.

**Требования:**
- Скелетоны (skeleton loaders) при загрузке данных
- Пустые состояния (empty states) с иконками и подписями
- Тосты/уведомления при ошибках API
- Отображение статуса подключения к Bitrix24
- Retry-логика при сбоях сети

### 3.5 Реальные данные в KPI-изменениях
**Текущее состояние:** Проценты изменений (`+18.2%`) захардкожены в HTML.

**Требования:**
- Рассчитывать изменения на бэкенде: текущий период vs предыдущий
- Динамически отображать стрелку вверх/вниз и цвет (зелёный/красный)
- Формат: `+12.5% к прошлой неделе` / `-3.1% к прошлому месяцу`

### 3.6 Сортировка менеджеров
**Текущее состояние:** `<select>` для сортировки есть, но не работает.

**Требования:**
- Рабочая сортировка: по конверсии, по выручке, по количеству сделок
- Анимация перестановки элементов при смене сортировки

### 3.7 Табы «Принятые/Непринятые»
**Текущее состояние:** Табы переключают `active` класс, но не фильтруют данные.

**Требования:**
- Таб «Принятые» — отображать источники принятых лидов
- Таб «Непринятые» — отображать источники непринятых/проваленных лидов
- Фильтрация на бэкенде по `STATUS_ID` лида

### 3.8 Кнопка обновления
**Текущее состояние:** Анимация вращения + визуальный эффект, но не перезагружает данные с API.

**Требования:**
- При нажатии — полная пересборка данных с Bitrix24 API
- Индикатор загрузки на время запроса
- Cooldown 5 секунд для предотвращения спама

### 3.9 Пагинация в таблице сделок
**Текущее состояние:** Отображается только 10 сделок (slice(0, 10) на бэкенде).

**Требования:**
- Пагинация: 10 / 25 / 50 записей на страницу
- Кнопки навигации по страницам
- Сортировка по колонкам (сумма, дни в работе, дата)
- Поиск по названию сделки

### 3.10 Кликабельные элементы → Bitrix24
**Требования:**
- Клик по сделке → открытие карточки сделки в Bitrix24 (`BX24.openApplication` или deep link)
- Клик по менеджеру → профиль пользователя в Bitrix24
- Клик по лиду → карточка лида

#### 4.2.3 UI/UX онбординга
- **Overlay** — затемнённый фон с перфорацией (spotlight) вокруг целевого элемента
- **Tooltip-облачко** — появляется рядом с элементом (сверху/снизу/сбоку, автопозиционирование)
- **Кнопки навигации:** «Назад», «Далее», «Пропустить тур», индикатор прогресса (шаг X из Y)
- **Анимация перехода** — плавное перемещение spotlight между элементами (300ms)
- **Пульсирующая точка** — на целевом элементе для привлечения внимания
- **Прогресс-бар** — визуальный индикатор прохождения тура в верхней части подсказки

#### 4.2.4 Техническая реализация
- Собственная реализация без внешних библиотек (чистый JS + CSS)
- Хранение статуса прохождения: `localStorage.setItem('onboarding_completed', 'true')`
- CSS: `.onboarding-overlay`, `.onboarding-spotlight`, `.onboarding-tooltip`
- Автоматический скролл к целевому элементу при переходе между шагами
- Адаптация под Bitrix24 iframe-режим (пропуск шагов, связанных с sidebar)
- Повторный запуск: кнопка «Пройти обучение» в разделе «Настройки» или по `?tour=true` в URL

#### 4.2.5 Условия запуска
- **Автозапуск:** если `localStorage` не содержит `onboarding_completed`
- **Ручной запуск:** пункт меню «Настройки → Пройти обучение заново»
- **Пропуск в iframe:** если `bitrix-mode`, скрывать шаги, связанные с sidebar
- **Задержка старта:** 1.5 секунды после загрузки данных, чтобы дашборд успел отрисоваться

#### 4.3.3 UI/UX подсказок
- **Иконка:** круглая кнопка 24×24px с символом `?`, цвет `var(--text-tertiary)`, при наведении — `var(--accent-primary)`
- **Расположение:** в `chart-header` каждого виджета, справа от заголовка (после `<h3>`)
- **Всплывающее окно (popover):**
  - Максимальная ширина: 320px
  - Фон: `var(--bg-card)` с тенью `var(--shadow-lg)`
  - Скруглённые углы: `var(--radius-lg)`
  - Заголовок виджета жирным шрифтом
  - Текст подсказки обычным шрифтом
  - «Стрелка» (треугольник) указывающая на иконку
  - Кнопка закрытия (×) в правом верхнем углу
  - Ссылка «Подробнее в базе знаний →» ведущая на соответствующую статью
- **Поведение:**
  - Клик по `❓` → появление popover с анимацией fadeIn (200ms)
  - Клик вне popover или по `×` → закрытие с fadeOut
  - Только один popover открыт одновременно
  - Автопозиционирование: если не хватает места справа — показывать слева

#### 4.3.4 Техническая реализация
- Единый компонент `HelpTooltip(targetElement, title, text, knowledgeBaseLink)`
- CSS-классы: `.help-btn`, `.help-popover`, `.help-popover-arrow`, `.help-popover-close`
- Данные подсказок хранятся в JS-объекте (словарь `helpTexts`)
- Popover создаётся динамически и добавляется в `document.body`
- Z-index: 500 (ниже модалок, выше контента)
- Для KPI-карточек — иконка в заголовке `.kpi-content` после `.kpi-label`
